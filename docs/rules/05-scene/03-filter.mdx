# 过滤器
## 使用场景
### 在toolbar中使用


#### 单行过滤器

- 过滤器较少时使用

- 过滤器和按钮合并在同一行显示，查询按钮紧挨着过滤器，导出或其他按钮放在最右边

- 不显示`label`，占位符中需要写上过滤器的名称

<img src="./03/单行过滤器.png" style={{width: "100%"}}></img>

#### 多行以滤器

- 单独作为一级容器时，显示内边距(`base`)， 显示背景色

- 过滤器区域布局方式：grid布局

- 查询、导出或其他按钮放在过滤器功能区， 规定过滤器功能区域最多只放2个普通按钮+2个自定义图标按钮`v1.1`


<img src="./03/多行过滤器.png" style={{width: "100%"}}></img>

#### 混合使用

情景一、多行过滤器功能按钮都放在功能条中

- 过滤器区域使用`多行行过滤器规范`

<img src="./03/情景1.png" style={{width: "100%"}}></img>

情景二、单行过滤器加少量功能按钮

- 过滤器区域使用`单行过滤器规范`

<img src="./03/情景2.png" style={{width: "100%"}}></img>

情景三、单行过滤器加较多功能按钮

- 过滤器区域使用`单行过滤器规范`

- 可以作为一级容器，显示内边距(`base`)， 显示背景色

<img src="./03/情景3.png"></img>

情景四、如果有额外显示信息可以放在功能条，比如统计数字等

#### 开发指导

- 以上情景都包裹在toolbar中

- 工具条可以使用`flex-line`组件进行布局，使用时打开`flex-line`的padding留出内边距

- 单行过滤器

布局可以直接摆放过滤器组件，给固定宽度即可

元素之间使用`el-space`留出间距

- 多行过滤器

布局方式使用grid响应式布局，可参照一下代码实现`:xs="24" :sm="12" :md="8" :lg="6" :xl="4"`；
过滤器内部元素布局使用`filter-item`组件，`filter-item`可以留出组件之间的间隙

多行布局时，应该考虑各列之间纵向对齐



### panel中使用
<img src="./03/panel过滤器.png" style={{width: "100%"}}></img>

#### 开发指导

- 过滤器放在panel的工具栏，直接给组件宽度即可

- 不超过3个过滤器时使用，可以加查询图标按钮

- 过滤组件要显示占位符（`placeholder`），要写清楚这个组件查询的内容项
